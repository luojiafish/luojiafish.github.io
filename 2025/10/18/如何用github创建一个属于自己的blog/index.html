常见问题速查

  • hexo d 报错：未找到 hexo-deployer-git

  • 推送被拒（push declined due to repository rule violations / secret 检测）
    GitHub 近期会自动扫描敏感信息。按提示点击日志中的链接 “To push, remove secret from commit(s) or allow the push”，确认允许后再部署。更稳妥的做法是：不要把 .env、密钥等文件放进仓库。

  • 访问 404

    • 仓库名必须是 你的用户名.github.io；
    • _config.yml 的 url 必须为 https://你的用户名.github.io（确保 https:// 冒号不漏）；
    • 部署分支应为 main，并在仓库 Settings → Pages 确认 Source 已指向该分支（通常用户页会自动识别）。
